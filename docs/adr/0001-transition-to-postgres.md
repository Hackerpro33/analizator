# ADR-0001: Переход с файлового хранилища на Postgres

- Статус: Proposed
- Дата: 2024-05-18

## Контекст

Текущее файловое хранилище ограничивает возможности совместного доступа, транзакционности и
масштабирования. Поддержка аналитики и очередей задач усложняется без централизованной БД.

## Решение

- Использовать Postgres как основную СУБД.
- Определить модели данных на SQLModel/SQLAlchemy и поддерживать миграции через Alembic.
- Настроить docker-compose сервис `postgres` и переменные окружения в `.env`.
- Обновить слой доступа к данным, чтобы избавиться от зависимостей от локальной файловой системы.

## Последствия

Положительные:

- Транзакционность и согласованность данных.
- Улучшенная схема для аналитики и агрегированных запросов.
- Готовность к горизонтальному масштабированию API и воркеров.

Негативные:

- Необходимость администрирования Postgres.
- Увеличение сложности деплоя и миграций.
- Требуются дополнительные ресурсы на CI для поднятия базы данных.

## План внедрения

1. Описать модели данных и миграции.
2. Создать слой репозиториев для CRUD-операций.
3. Настроить фикстуры баз данных в тестах и обновить контрактные тесты API.
4. Обновить документацию и скрипты развёртывания.
